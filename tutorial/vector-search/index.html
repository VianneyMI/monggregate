    üîç Vector Search with Monggregate

    MongoDB Atlas provides powerful vector search capabilities through the $vectorSearch stage, enabling approximate nearest neighbor (aNN) search on vector embeddings. Monggregate makes these advanced vector search features accessible through an intuitive Python interface.

    üìã What is Vector Search?

    üí° Vector search allows you to find documents with similar vector embeddings to a query vector, enabling semantic search, recommendations, and AI-powered applications.

    Atlas Vector Search offers:

    • üß† Semantic similarity search using vector embeddings
    • üîç Approximate nearest neighbor (aNN) algorithms for efficient vector comparison
    • ‚ö° Fast retrieval of similar items from large collections
    • üß© Pre-filtering to narrow search scope and improve relevance
    • üîÑ Integration with AI models like OpenAI, Hugging Face, and others

    üìö Vector search is particularly useful for applications like: - Semantic text search that understands meaning, not just keywords - Image similarity search - Recommendation systems - AI-powered chatbots and RAG (Retrieval Augmented Generation)

    üî∞ Prerequisites for Vector Search

    Before using vector search with Monggregate, you need to:

    1. üìä Create an Atlas Vector Search index on your collection
    2. üß™ Generate vector embeddings for your documents using an embedding model
    3. üíæ Store these embeddings in your MongoDB documents

    ‚ö†Ô∏è Vector search is only available on MongoDB Atlas clusters running v6.0.11 or v7.0.2 and later.

    from monggregate import Pipeline

# Generate or obtain your query vector (embedding)
query_vector = [0.1, 0.2, 0.3, 0.4, ...]  # Your vector dimensions here

# Build the vector search pipeline
pipeline = Pipeline()
pipeline.vector_search(
    index="vector_index",        # Name of your Atlas Vector Search index
    path="embedding",            # Field containing vector embeddings
    query_vector=query_vector,   # Your search vector
    num_candidates=100,          # Number of candidates to consider
    limit=10                     # Number of results to return
)

    üìò This query will find the 10 documents whose embedding vectors are most similar to your query vector, considering 100 nearest neighbors during the search.

    path="product_vector",

    import numpy as np
from sentence_transformers import SentenceTransformer
from monggregate import Pipeline
import pymongo

# Connect to MongoDB
client = pymongo.MongoClient("mongodb+srv://...")
db = client["knowledgebase"]

# Load embedding model
model = SentenceTransformer('all-MiniLM-L6-v2')

# Generate embedding for user query
user_query = "How do I implement authentication in my application?"
query_embedding = model.encode(user_query).tolist()

# Create vector search pipeline
pipeline = Pipeline()
pipeline.vector_search(
    index="document_vectors",
    path="embedding",
    query_vector=query_embedding,
    num_candidates=100,
    limit=5,
    filter={
        "document_type": "article",
        "status": "published"
    }
).project(
    title=1,
    content=1,
    url=1,
    score={"$meta": "vectorSearchScore"}
)

# Execute search
results = list(db.documents.aggregate(pipeline.export()))

# Display results
for doc in results:
    print(f"Title: {doc['title']}")
    print(f"Score: {doc['score']:.4f}")
    print(f"URL: {doc['url']}")
    print("-" * 40)

    üî¨ Technical Details

    • üî¢ Vector dimensions: Your query vector must have the same number of dimensions as the vectors in your indexed field
    • üéØ numCandidates: Should be greater than the limit for better accuracy, typically 10-20x for optimal recall
    • ‚ö° Performance tuning: Adjust numCandidates to balance between search quality and speed
    • üîÑ Filtering: Only works on indexed fields marked as the "filter" type in your vector search index
    • üìä Scoring: For cosine and dotProduct similarities, scores are normalized using the formula: score = (1 + cosine/dot_product(v1,v2)) / 2
